기본적으로 싱글스레드인 Node js가 CPU 코어를 모두 사용할 수 있게 해주는 모듈

- 포트를 공유하는 노드 프로세스를 여러 개 둘 수 있다.
- 요청이 많이 들어왔을때 병렬로 실행된 서버의 개수만큼 요청이 분산된다.
- 서버에 무리가 덜간다.
- 코어가 8개인 서버가 있을때 (보통은 코어 1개만 활용한다.)
- Cluster로 코어 하나당 Node js 프로세스 하나를 배정 가능하다.
- 성능이 8배가 되는 것은 아니지만 개선된다.
- 단점 : 컴퓨터 자원(메모리, 세션 등등) 공유하지 못한다.
- 단점은 Redis 등 별도 서버로 해결
