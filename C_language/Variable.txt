변수 생존시간에 따른 변수 분류
- 정적할당 (Static)
    프로그램이 종료되면 저장공간이 반납되는 방법
- 자동할당 (Auto)
    블록이 종료되면 저장공간이 반납되는 방법
    Auto의 경우 stack에 쌓이기 때문에 빠르다.

변수 선언 위치에 따른 분류
- 전역변수
    - 정적할당
- 지역변수
    - 기본적으로 자동할당
    - 하지만 저장 유형 지정자로 정적할당 가능
        - 저장 유형 지정자
            1. Auto : 생략이 가능
                ex) auto int i = 0; == int i = 0;
            2. Register : 생존기간이 Auto와 같다.
                - 지역변수로만 사용가능
                - CPU 레지스터에 저장
                - 처리 속도가 빠르다.
                - 제한사항
                    1. 사용시 정수, 문자형으로만 사용 가능
                    2. 변수 개수에 제한이 있다. (일반적으로 3개)
                    3. 주소 연산자(&) 사용 불가능
            3. Static : 프로그램 시작때 1번 초기화 된다.
            4. Extern : 함수 밖에서 선언된 변수 (전역변수)
                - 파일 외부에서도 사용가능
                - 프로그램 종료될때 반환된다.
                - 저장공간이 할당되지 않는다.
